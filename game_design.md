# 春秋战国探险游戏设计文档

## 游戏概述
这是一个基于春秋战国时期的文字类探险游戏，玩家扮演一个出身于秦国小镇的探险小队，通过探索大世界来获取资源、发现城市和完成任务。

## 核心系统

### 1. 城镇系统
- **初始城镇**：秦国的一个小镇，包含以下建筑物
  - 兵器库：购买和升级装备
  - 驿站：出门探索
  - 市集：购买粮食、资源等
  - 酒馆：招募队员、获取情报
  - 民居：休息恢复体力

### 2. 探索系统
- **探索机制**：通过驿站选择探索方向和距离
- **消耗**：每次探索消耗粮食，消耗数量与探索距离成正比
- **探索结果**：
  - 发现新城市
  - 遭遇随机事件（战斗、宝藏、自然灾害等）
  - 获得资源

### 3. 大世界系统
- **坐标系**：使用二维坐标系统 (X, Y)
- **城市生成**：随机生成60个城市，分布在合理的坐标位置
- **城市属性**：
  - 名称
  - 所属国家
  - 规模（小型、中型、大型）
  - 特产
  - 友好度

### 4. 资源系统
- **粮食**：探索的基本消耗品
- **金钱**：用于购买装备和资源
- **装备**：影响探索效率和战斗能力
- **队员**：增加探险队的能力

## 城市生成算法

### 坐标生成规则
1. 以咸阳为中心（X:100, Y:100）
2. 其他城市围绕咸阳分布
3. 城市之间保持合理距离（至少20单位）
4. 坐标范围：X [0, 200], Y [0, 200]

### 城市属性生成
1. **国家分配**：根据春秋战国时期的主要国家（秦、楚、齐、燕、赵、魏、韩）随机分配
2. **规模**：根据城市与咸阳的距离，近的城市规模较大
3. **特产**：根据国家和地理位置随机生成
4. **友好度**：初始友好度根据国家关系设定

## 游戏流程
1. 玩家在初始城镇准备（购买粮食、装备、招募队员）
2. 通过驿站选择探索方向和距离
3. 系统计算探索结果（消耗粮食、发现城市、遭遇事件）
4. 玩家可以选择在发现的城市进行交易、休息等活动
5. 重复探索过程，扩大地图范围

## 技术实现
- **游戏状态**：使用JSON存储游戏状态
- **城市数据**：预生成60个城市的坐标和属性
- **随机事件**：基于概率的随机事件系统
- **用户界面**：文字菜单式界面，通过命令行或简单GUI实现
